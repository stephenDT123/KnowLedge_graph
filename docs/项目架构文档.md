# 本科《Python程序设计》教育知识图谱项目架构文档

## 项目概述
本项目旨在为本科《Python程序设计》课程构建一个基于GraphRAG的教育知识图谱系统，集成GraphXR实现多维可视化分析。项目从传统RAG方案演进为GraphRAG方案，提供更强大的检索、摘要和可解释问答能力。

## 技术架构

### 1. 数据层
- **多模态数据源**：
  - PDF教材：中文教材PDF，支持文本提取和OCR
  - 视频课程：慕课、精品课程视频
  - 课程大纲：教学目标、知识点结构
  - 评测资源：作业、实验、测试题

### 2. 处理层
- **PDF处理模块** (`pdf_data_processor.py`)：
  - 支持PyMuPDF和PyPDF2双引擎文本提取
  - 中文分词（jieba）和停用词过滤
  - 可选OCR功能（pytesseract）
  - 文本规范化处理

- **GraphRAG构建模块** (`graphrag_builder.py`)：
  - **LLM API集成**：支持OpenAI、Anthropic、Azure等多种LLM提供商
  - **Embedding模型**：集成文本嵌入向量生成，支持批量处理
  - **文档切分**：智能句子边界切分，支持重叠块
  - **实体抽取**：基于LLM的教育领域术语识别（概念、技能、评测、资源）
  - **关系抽取**：LLM驱动的先修、包含、依赖等关系识别
  - **社区发现**：基于图连通分量的知识社区聚类
  - **摘要生成**：LLM驱动的社区结构化摘要报告

### 3. 存储层
- **中间格式**：JSONL格式存储GraphRAG各阶段数据
- **图数据库**：计划集成Neo4j存储RDF三元组
- **向量索引**：为文本单元建立向量索引支持语义检索

### 4. 服务层
- **FastAPI框架** (`main.py`)：
  - 提供RESTful API接口
  - 支持知识图谱查询和检索
  - 计划集成GraphRAG检索服务

### 5. 可视化层
- **GraphXR集成**：
  - 多维可视化看板
  - 支持先修路径、难度进阶、掌握度、评测覆盖等维度
  - 交互式图谱探索

## GraphRAG完整架构

### 1. 配置系统 (`config.py`)
- **LLM配置**：支持OpenAI、Anthropic、Azure、Google、DeepSeek等多种提供商
- **Embedding配置**：支持OpenAI、Azure、Cohere、Voyage等嵌入模型
- **环境变量支持**：通过环境变量动态配置API密钥和模型参数
- **默认配置**：提供合理的默认值，简化项目启动

### 2. 客户端抽象层
- **LLM客户端** (`LLMClient`)：抽象LLM调用接口
  - `OpenAIClient`：OpenAI API实现
  - 可扩展支持Anthropic、Azure等客户端
- **Embedding客户端** (`EmbeddingClient`)：抽象嵌入向量生成
  - `OpenAIEmbeddingClient`：OpenAI Embedding API实现
  - 支持批量处理，优化API调用效率

### 3. 数据模型层
- **Document**：文档元数据和原始文本
- **TextUnit**：文本切分单元，包含嵌入向量
- **Entity**：教育领域实体，支持概念、技能、评测、资源等类型
- **Relationship**：实体间关系，包含证据和权重
- **Community**：知识社区，包含实体聚类和摘要
- **GraphRAGDataset**：完整数据集封装

### 4. 核心构建流程

#### 阶段1：数据准备与文档构建
```
原始PDF/视频 → 文本提取 → 中文分词 → 停用词过滤 → 规范化文本
↓
Document对象（包含元数据和原始文本）
```

#### 阶段2：文本单元切分与嵌入
```
Document文本 → 智能句子边界切分 → 重叠文本块
↓
TextUnit对象（包含文本、分词、嵌入向量）
```

#### 阶段3：实体关系抽取（LLM驱动）
```
TextUnit文本 → LLM实体识别 → 实体统计与分类
↓
Entity对象（包含类型、频率、描述、嵌入向量）
↓
LLM关系抽取 → 关系证据收集 → 关系权重计算
↓
Relationship对象（包含源、目标、类型、权重、证据）
```

#### 阶段4：社区发现与摘要生成
```
实体关系图 → 连通分量分析 → 社区聚类
↓
Community对象（包含实体列表、关系列表、标题）
↓
LLM社区摘要生成 → 教学建议 → 学习路径推荐
↓
结构化社区报告
```

#### 阶段5：数据持久化与索引
```
所有数据对象 → JSONL序列化 → 文件存储
↓
配置信息 → JSON保存 → 版本管理
↓
嵌入向量 → 向量索引（为检索优化）
```

### 5. 异步处理架构
- **异步API调用**：使用asyncio优化LLM和Embedding API调用
- **批量处理**：支持批量嵌入向量生成，减少API调用次数
- **错误处理**：优雅的异常处理和降级机制
- **进度反馈**：实时构建进度显示

### 6. 命令行接口
- **参数解析**：支持PDF/视频路径、输出目录、配置参数
- **环境配置**：从环境变量加载API密钥和模型设置
- **进度显示**：实时显示构建进度和统计信息
- **结果输出**：结构化JSON摘要和完整数据集保存

## 核心数据模型

### 1. Document（文档）
```python
@dataclass
class Document:
    id: str          # 文档唯一标识
    title: str       # 文档标题
    source_type: str # 来源类型（pdf/video）
    source_path: str # 源文件路径
    text: str        # 完整文本内容
```

### 2. TextUnit（文本单元）
```python
@dataclass
class TextUnit:
    id: str          # 文本单元ID
    document_id: str # 所属文档ID
    index: int       # 在文档中的序号
    text: str        # 文本内容
    tokens: List[str] # 分词结果
```

### 3. Entity（实体）
```python
@dataclass
class Entity:
    id: str          # 实体ID
    name: str        # 实体名称
    entity_type: str # 实体类型（Concept/Skill/Assessment等）
    frequency: int   # 出现频次
    description: str # 描述信息
    document_ids: List[str]    # 出现文档
    text_unit_ids: List[str]   # 出现文本单元
```

### 4. Relationship（关系）
```python
@dataclass
class Relationship:
    id: str          # 关系ID
    source: str      # 源实体ID
    target: str      # 目标实体ID
    relation_type: str # 关系类型
    weight: int      # 关系权重
    evidence_text_unit_ids: List[str] # 证据文本单元
```

## 教育领域本体设计

### 核心实体类型
- **Concept（概念）**：变量、数据类型、运算符、函数、类等
- **Skill（技能）**：调试、编程、算法设计等
- **Assessment（评测）**：实验、作业、测试等
- **Resource（资源）**：案例、示例、视频、教材等
- **LearningObjective（学习目标）**：课程教学目标

### 关键关系类型
- **PREREQUISITE**：先修关系（A是B的基础）
- **DEPENDS_ON**：依赖关系（A依赖B）
- **CONTAINS**：包含关系（A包括B）
- **USED_FOR**：用途关系（A用于B）
- **IMPLEMENTS**：实现关系（A实现B）

### 运行示例

#### 方式1：使用默认配置
```python
# 构建GraphRAG数据集
from models.graphrag_builder import build_graphrag_dataset

dataset = await build_graphrag_dataset(
    output_dir="models/graphrag_output",
    pdf_paths=["models/附件：学员操作手册.pdf"]
```

#### 方式2：命令行接口
```bash
# 设置环境变量
$env:OPENAI_API_KEY = "your-api-key"

# 运行构建
python models/graphrag_builder.py --pdf "models/附件：学员操作手册.pdf" --output "graphrag_output"
```

#### 方式3：自定义配置
```python
from models.config import GraphRAGConfig, LLMConfig, EmbeddingConfig

# 创建自定义配置
config = GraphRAGConfig(
    llm_config=LLMConfig(
        provider="openai",
        model="gpt-4o-mini",
        temperature=0.1
    ),
    embedding_config=EmbeddingConfig(
        provider="openai",
        model="text-embedding-3-small"
    ),
    chunk_size=500,
    chunk_overlap=50
)

# 使用自定义配置构建
dataset = await build_graphrag_dataset(
    output_dir="custom_output",
    pdf_paths=["data/textbook.pdf"],
    config=config
)
```
